# Luna 表达式解析器与条件注入设计

> **版本**: v1.0
> **更新日期**: 2026-03-28
> **项目定位**: 条件注入功能的核心组件，负责将用户输入的条件表达式字符串解析为可执行的抽象语法树（AST），并生成对应的Java字节码

---

## 1. 功能需求

### 1.1 核心功能

| 功能        | 说明                                  |
|-----------|-------------------------------------|
| **变量访问**  | 局部变量、方法参数、对象字段、`this` 引用            |
| **条件输出**  | 仅当满足特定逻辑（如 `${userId == 100}`）时输出日志 |
| **字节码生成** | 高性能字节码生成，最小化性能影响                    |

### 1.2 支持的表达式语法

#### 变量访问

| 语法格式   | 示例                        | 说明              |
|--------|---------------------------|-----------------|
| 局部变量   | `${count}`                | 访问方法内的局部变量      |
| 方法参数   | `${param[0]}` 或 `${arg0}` | 访问方法的第1个参数      |
| 字段访问   | `${user.name}`            | 访问对象的字段（支持链式访问） |
| this引用 | `${this.field}`           | 访问当前对象的字段       |
| 返回值    | `${return}`               | 访问方法返回值         |

#### 运算符

| 运算符类型 | 支持的运算符               | 示例                                     |
|-------|----------------------|----------------------------------------|
| 算术运算  | +, -, *, /, %        | `${a + b}`, `${count * 2}`             |
| 比较运算  | >, <, >=, <=, ==, != | `${age} > 18`, `${status} != "active"` |
| 逻辑运算  | &&, \|\|, !          | `${a && b}`, `${!flag}`                |
| 括号    | ( )                  | `${(a > b) && (c < d)}`                |

#### 字面量

| 类型   | 示例                   |
|------|----------------------|
| 整数   | `18`, `-5`, `0xFF`   |
| 浮点数  | `3.14`, `-2.5`       |
| 字符串  | `"hello"`, `'world'` |
| 布尔值  | `true`, `false`      |
| null | `null`               |

### 3.2 Token类定义

```java
public class Token {
    private final TokenType type;
    private final String lexeme;
    private final Object literal;
    private final int position;

    public Token(TokenType type, String lexeme, Object literal, int position) {
        this.type = type;
        this.lexeme = lexeme;
        this.literal = literal;
        this.position = position;
    }

    public TokenType getType() { return type; }
    public String getLexeme() { return lexeme; }
    public Object getLiteral() { return literal; }
    public int getPosition() { return position; }
}
```

## 4. 语法分析设计

### 4.1 递归下降解析算法

```java
public class ConditionalExpressionParser {

    // 解析入口
    public ExpressionNode parse(String expression) {
        this.tokens = tokenize(expression);
        this.current = 0;
        return parseLogicalOr();
    }

    // 逻辑或 (最低优先级)
    private ExpressionNode parseLogicalOr() {
        ExpressionNode left = parseLogicalAnd();
        while (match("||")) {
            ExpressionNode right = parseLogicalAnd();
            left = new LogicalExpressionNode(left, "||", right);
        }
        return left;
    }

    // 逻辑与
    private ExpressionNode parseLogicalAnd() {
        ExpressionNode left = parseComparison();
        while (match("&&")) {
            ExpressionNode right = parseComparison();
            left = new LogicalExpressionNode(left, "&&", right);
        }
        return left;
    }

    // 比较运算 (最高优先级)
    private ExpressionNode parseComparison() {
        ExpressionNode left = parsePrimary();

        if (match(">")) return new ComparisonNode(left, parsePrimary(), ">");
        if (match("<")) return new ComparisonNode(left, parsePrimary(), "<");
        if (match("==")) return new ComparisonNode(left, parsePrimary(), "==");
        if (match("!=")) return new ComparisonNode(left, parsePrimary(), "!=");

        return left;
    }
}
```

## 5. AST节点设计

### 5.1 节点接口

```java
public interface ExpressionNode {
    Object evaluate(Map<String, Object> variables);
    void generateBytecode(MethodVisitor mv, Map<String, Integer> varSlots);
}
```

### 5.2 变量访问节点

```java
public class VariableNode implements ExpressionNode {
    private final String variableName;

    @Override
    public Object evaluate(Map<String, Object> variables) {
        return variables.get(variableName);
    }

    @Override
    public void generateBytecode(MethodVisitor mv, Map<String, Integer> varSlots) {
        Integer slot = varSlots.get(variableName);
        mv.visitVarInsn(Opcodes.ALOAD, slot);
    }
}
```

### 5.3 比较运算节点

```java
public class ComparisonNode implements ExpressionNode {
    private final ExpressionNode left;
    private final ExpressionNode right;
    private final String operator;

    @Override
    public Object evaluate(Map<String, Object> variables) {
        Object leftVal = left.evaluate(variables);
        Object rightVal = right.evaluate(variables);

        switch (operator) {
            case ">": return compare(leftVal, rightVal) > 0;
            case "<": return compare(leftVal, rightVal) < 0;
            case "==": return Objects.equals(leftVal, rightVal);
            case "!=": return !Objects.equals(leftVal, rightVal);
            default: throw new IllegalArgumentException("不支持的操作符: " + operator);
        }
    }
}
```

## 6. 冲突检测机制

### 6.1 语法冲突检测

```java
public class ExpressionConflictDetector {

    /**
     * 语法验证：括号匹配、变量格式、运算符合法性
     */
    public static ValidationResult validateSyntax(String expression) {
        // 1. 括号匹配检查
        if (!isBracketsBalanced(expression)) {
            return ValidationResult.error("括号不匹配");
        }

        // 2. 变量格式验证: ${variableName}
        if (!isValidVariableFormat(expression)) {
            return ValidationResult.error("变量格式错误，正确格式: ${variableName}");
        }

        // 3. 运算符合法性检查
        if (!isValidOperators(expression)) {
            return ValidationResult.error("包含非法运算符");
        }

        return ValidationResult.success();
    }

    /**
     * 检查括号是否平衡
     */
    private static boolean isBracketsBalanced(String expression) {
        Stack<Character> stack = new Stack<>();
        for (char ch : expression.toCharArray()) {
            if (ch == '(') {
                stack.push(ch);
            } else if (ch == ')') {
                if (stack.isEmpty() || stack.pop() != '(') {
                    return false;
                }
            }
        }
        return stack.isEmpty();
    }
}
```

### 6.2 语义冲突检测

```java
public class SemanticConflictChecker {

    public static ValidationResult checkMethodCompatibility(
            String expression, MethodInfo methodInfo) {

        // 1. 提取所有变量
        Set<String> variables = extractVariables(expression);

        // 2. 检查变量在方法作用域中的存在性
        for (String var : variables) {
            if (!isVariableInScope(var, methodInfo)) {
                return ValidationResult.error(
                    "变量 '" + var + "' 在方法作用域中不存在");
            }
        }

        // 3. 类型兼容性检查
        for (String var : variables) {
            ValidationResult typeCheck = checkTypeCompatibility(var, methodInfo);
            if (!typeCheck.isValid()) {
                return typeCheck;
            }
        }

        return ValidationResult.success();
    }
}
```

### 6.3 运行时冲突检测

```java
public class RuntimeConflictDetector {

    public static ValidationResult checkInjectionConflicts(
            List<InjectionPoint> existingInjections,
            InjectionPoint newInjection) {

        // 1. 位置冲突检测
        for (InjectionPoint existing : existingInjections) {
            if (isSameLocation(existing, newInjection)) {
                ValidationResult conflict = checkConditionConflict(
                    existing.getCondition(),
                    newInjection.getCondition());
                if (conflict.hasConflict()) {
                    return conflict;
                }
            }
        }

        // 2. 性能影响评估
        return assessPerformanceImpact(existingInjections, newInjection);
    }
}
```

## 7. 安全沙箱设计

### 7.1 静态层

扫描埋点模板中的 Java 调用。禁止使用：

- 反射 (`reflect`)
- JNI 调用
- 核心破坏性 API（如 `System.exit()`）

### 7.2 动态层

| 限制类型     | 实现方式                        |
|----------|-----------------------------|
| **时间限制** | 由于注入代码是在业务线程跑，若解析耗时过长，会自动打断 |
| **资源配额** | 限制单次注入逻辑创建的对象总数             |
